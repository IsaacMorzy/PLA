## 3) Content Refinement Strategy (apply to every asset)

Use this 6-step refinement checklist before publish:

1. **Specificity pass**
   - Replace abstract claims with concrete outcomes.
   - Bad: “Built for trust.”
   - Better: “Donors can review campaign progress and wallet activity before giving.”

2. **Proof pass**
   - Add one verifiable signal: on-chain visibility, donor activity, progress-to-goal, campaign wallet linkage.

3. **Audience pass**
   - State who the message is for (donor, creator, partner).
   - Remove internal/design-meta phrasing.

4. **Action pass**
   - One CTA only per asset; no mixed asks.
   - Preferred CTAs: **Donate now**, **Browse campaigns**, **Launch campaign**.

5. **Compression pass**
   - Tighten by 15–25% while preserving proof and intent.

6. **Repurpose pass**
   - Extract 3 short variants from each long-form piece:
     - headline hook
     - proof line
     - CTA line
