# Sitewide Content Refresh Plan (No-Genericism Pass)

## Why this plan
PeaceLeague Africa already has strong UI direction; this pass upgrades **message precision** so every page sounds specific, credible, and conversion-oriented. We will remove vague/meta phrasing (e.g., “this page should feel…”) and replace it with concrete donor, creator, and partner outcomes.

## Business goals
1. Increase donation intent from first-session visitors.
2. Increase campaign creation completion confidence.
3. Improve trust comprehension (traceability, fee model, verification flow).
4. Strengthen content quality for SEO + social repurposing.

## Audience priorities
1. **Donors**: need confidence, urgency clarity, and proof.
2. **Campaign creators**: need launch guidance + legitimacy framing.
3. **Partners/media**: need narrative clarity and proof of operation.

## Voice + copy rules (applied globally)
- Replace abstract/meta copy with observable outcomes.
- Use plain language with concrete nouns and verbs.
- Prioritize user value over design self-reference.
- Keep trust claims specific: on-chain visibility, campaign wallet activity, no platform fee.
- Keep CTA verbs consistent: **Donate now**, **Browse campaigns**, **Launch campaign**.

## Funnel mapping by page group
- **Awareness**: `/`, `/about`, `/how-it-works`, `/blog`, `/image-gallery`, `/events`
- **Consideration**: `/campaigns`, `/campaign/[slug]`, `/faq`, `/testimonials`, `/donor-list`
- **Decision**: `/create`, `/donation`, `/contact`, `/support-ticket`
- **Retention/ops**: `/dashboard`, `/careers`, legal pages

## Execution phases

### Phase A — Flagship conversion pages
- `/` + `home-client.tsx`
- `/campaigns`
- `/campaign/[slug]`
- `/create`

### Phase B — Trust and consideration pages
- `/faq`, `/testimonials`, `/donor-list`, `/about`, `/how-it-works`

### Phase C — Support and utility pages
- `/donation`, `/contact`, `/support-ticket`, `/dashboard`, `/events`, `/careers`, `/image-gallery`

### Phase D — Editorial and legal polish
- `/blog`, `/blog/[slug]`, `/privacy-policy`, `/terms-conditions`, error/empty states

## Page-level checklist
For each route:
1. Remove meta-design language (“this page should feel…”).
2. Replace with specific user intent and action language.
3. Ensure hero description answers: who, what, why now.
4. Ensure CTA is concrete and stage-appropriate.
5. Keep internal consistency with shared CTA/trust copy.

## Repurposing outputs generated by this pass
- Homepage hero + trust blocks → LinkedIn narrative post + email intro.
- How-it-works steps → short X thread + onboarding snippet.
- FAQ trust answers → support macros + sales/partner FAQ.
- Campaign detail trust copy → donation retargeting email sections.

## QA gates
- Copy specificity audit (no generic/meta phrasing).
- CTA consistency audit.
- Readability + contrast already covered by frontend passes.
- `pnpm -s exec tsc --noEmit`.

## Success criteria
- No major route contains self-referential design copy.
- All key page intros describe user outcomes, not layout intent.
- CTAs and trust language remain consistent sitewide.
